六 树

6.1 树的定义与概念

6.1.1 树的定义

树（TREE） 使n(n≥0) 个结点的有限集，n=0时称为空树。在任意一颗非空树中：
  1. 有且只有一个特定的称为根（Root）的结点；
  1. 当n>1时，其余结点可分为m(m>0)个不相交的有限集T1,T2……Tn，其中每一个集合本身又是一棵树，并且称为根的子树（SubTree）。

6.1.2 结点分类

结点拥有的子树数量称为结点的度（Degree）。

度为0的结点称为叶结点（Leaf）或终端结点；

度不为0的结点称为非终端结点或分支结点。

除根节点之外，分支结点也称为内部结点。树的度是树内各结点的度的最大值。

6.1.3 结点间关系

结点的子树的根称为该结点的孩子（Child），相应地，该结点称为孩子的双亲（Parent）。

同一个双亲的孩子之间互称兄弟（Sibling）。

结点的祖先 是从根到该结点所经分支上的所有结点。

以某结点为根的子树中的任一结点都称为该结点的子孙 。

6.1.4 层次&深度&森林

结点层次（Level）从根开始定义，根为第一层，根的孩子为第二层。

其双亲在同一层的结点互为堂兄弟。

树中结点的最大层次称为树的深度（Depth）或高度。

森林（Forest）是m（m≥0）颗互不相交的树的集合。

6.1.5 线性表与树的结构对比

线性结构树结构
第一个数据元素：无前驱根结点：无双亲，唯一
最后一个数据元素：无后继叶结点：无孩子，可以多个
中间元素：一个前驱，一个后继中间结点：一个双亲多个孩子

6.4 二叉树

6.4.1 二叉树的定义

二叉树（Binary Tree）是 n（n≥0）个结点的有限集合，该集合或者为空集（称为空二叉树），或者由一个根结点和两颗互不相交的、分别称为根结点的左子树和右子树的二叉树组成。

6.4.2 二叉树的特点

❕二叉树特点：

  • 每个结点最多有两颗子树。
  • 左子树和右子树是由顺序的，不能任意颠倒。
  • 即使有一颗子树也得区分左右。

❕二叉树基本形态

  • 空二叉树。
  • 只有一个根结点。
  • 根结点只有左子树
  • 根结点只有右子树
  • 根结点既有左子树又有右子树

6.4.3 特殊二叉树

  1. 斜树
    斜树 所有的结点都只有左子树或者只有右子树。
  1. 满二叉树
    满二叉树 二叉树所有的分支结点都存在左子树和右子树，并且所有叶子都在同一层上。

❕满二叉树特点：

  • 叶子只能在最下层。
  • 非叶子结点的度一定为2。
  • 同样深度的二叉树，满二叉树结点树最多，叶子数最多。

  1. 完全二叉树
    对一颗具有n个结点的二叉树按层序编号，如果编号为i（1≤i≤n）的结点与同样深度的满二叉树中编号为i的结点在二叉树中的位置完全相同，则这颗二叉树称为完全二叉树。

❕完全二叉树特点

  • 叶子结点只能在下面两层。
  • 最下层的叶子一定集中在左部连续位置。
  • 倒数二层，若有叶子结点，一定都在右部连续位置。
  • 如果结点度为1，则该结点只有左孩子，不存在只有右孩子的结点。
  • 同样结点的二叉树，完全二叉树的深度最小。

6.4.6 二叉树的遍历

二叉树的遍历（ traversing binary tree ）是指从根结点出发，按照某种 次序 依次 访问 二叉树中所有的结点，使得每个结点被访问一次且仅被访问一次。

💡 中序遍历序列加左右任意单边的遍历序列可以唯一确定一颗二叉树

  1. 前序遍历 - 深度优先
    规则是若二叉树为空，则空操作返回，否则先访问根结点，然后前序遍历左子树，再前序遍历右子树。

    遍历顺序：ABDGHCEIF

void preOrderTraverse(BiTree tree){
	if(tree == null){
		return; 
	}
	system.out.println(tree.val()); // 对结点的操作
	preOrderTraverse(tree.leftChild()); // 遍历左子树结点数据
	preOrderTraverse(tree.rightChild()); // 遍历右子树结点数据
}
  1. 中序遍历 - 深度优先
    规则是若树为空，则空操作返回，否则从根结点开始（注意并不是先访问根结点），中序遍历根结点的左子树，然后是访问根结点，最后中序遍历右子树。

    遍历顺序： GDHBAEICF

void preOrderTraverse(BiTree tree){
	if(tree == null){
		return; 
	}
	preOrderTraverse(tree.leftChild()); // 遍历左子树结点数据
	system.out.println(tree.val()); // 对结点的操作
	preOrderTraverse(tree.rightChild()); // 遍历右子树结点数据
}
  1. 后序遍历 - 深度优先
    规则是若树为空，则空操作返回，否则从左到右先叶子后结点的方式遍历访问左右子树，最后是访问根结点。

    遍历顺序：GHDBIEFCA

void preOrderTraverse(BiTree tree){
	if(tree == null){
		return; 
	}
	preOrderTraverse(tree.leftChild()); // 遍历左子树结点数据
	preOrderTraverse(tree.rightChild()); // 遍历右子树结点数据
	system.out.println(tree.val()); // 对结点的操作
}
  1. 层序遍历 - 广度优先
    规则是若树为空，则空操作返回，否则从树的第一层，也就是根结点开始访问，从上而下逐层遍历，在同一层中，按从左到右的顺序对结点逐个访问。

    遍历顺序：ABCDEFGHI

6.4.7 线索二叉树

指向前驱和后继的指针称为线索，加上线索的二叉链表称为线索链表，相应地二叉树称为 线索二叉树（Threaded Binary Tree）
对二叉树以某种次序遍历使其变为线索二叉树的过程称作是线索化。

如果所用的二叉树需要经常遍历或查找结点时需要某种遍历序列中的前驱和后继，

BiThrThree pre; /* 全局变量，始终指向上一访问结点 */
/* 中序遍历进行中序线索化 */
void InThreading(BiThrThree p)
{
	if(p)
	{
		InThreading(p->lchild);/* 递归左子树线索化 */
		if(!p->lchild)/* 没有左孩子 */
		{
			p->LTag=Thread;/* 前驱线索 */
			p->lchind=pre;/* 左孩子指针指向前驱 */
		}
		if(!pre->rchild)/* 前驱没有右孩子 */
		{
			pre->RTag=Thread;/* 后继线索 */
			pre->rchind=p;/* 前驱右孩子指针指向后继（当前结点p） */
		}
		pre=p;/* 保持pre 指向p的前驱 */
		InThreading(p->rchild);/* 递归右子树线索化 */
	}
}

6.4.8 赫夫曼树及其应用

从树中一个结点到另一个结点之间的分支构成两个结点之间的路径，路径上的分支数目称作路径长度。

树的路径长度就是从树根到每一结点的路径长度之和。

带权路径长度WPL最小的二叉树称为赫夫曼树。
