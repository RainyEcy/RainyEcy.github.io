Javac源码分析

Javac编译过程图

编译过程大致可以分为1个准备过程和三个处理过程：

  1. 初始化插入注解处理器
  1. 解析和填充符号表的过程：
    1. 词法、语法分析将源代码的字符流变为标记集合，构造出抽象语法树。
    1. 填充符号表。产生符号地址和符号信息
  1. 插入式注解处理器的注解处理过程：插入式注解处理器的执行阶段。
  1. 分析和字节码生成的过程
    1. 标注检查。对语法的静态信息进行检查。
    1. 数据流及控制流分析。对程序动态运行过程进行检查。
    1. 解语法糖。将简化代码编写的语法糖还原为原有的形式。
    1. 字节码生成。将前面各个步骤所生成的信息转化为字节码。

上述的三个处理过程里，执行注解处理器又可能产生新的符号，如果有新的符号产生，回到之前的解析阶段，重新处理这些新的符号。

Javac编译动作入口是com.sun.tools.javac.man.JavaCompiler.compile();

// 准备过程 1 初始化插入注解处理器
initProcessAnnotations(processors, sourceFileObjects, classnames);

// 处理过程 1.1 parseFiles 词法分析，语法分析
// 处理过程 1.2 enterTrees 输出到符号表
// 处理过程 2 processAnnotations 执行注解处理
processAnnotations(
  enterTrees(stopIfError(CompileState.PARSE,initModules(stopIfError(CompileState.PARSE, 
		parseFiles(sourceFileObjects))))),classnames);

// …………………… 忽略中间代码


case BY_TODO:
		// 处理过程 3.1 attribute 标注检查
		// 处理过程 3.2 flow 数据流控制流分析
		// 处理过程 3.3 desugar 解语法糖
		// 处理过程 3.4 生成最终字节码
    while (!todo.isEmpty())
        generate(desugar(flow(attribute(todo.remove()))));
    break;

解析与填充符号表 - 处理过程 1

词法分析、语法分析 - 处理过程 1.1 parseFiles

词法分析是将源代码的字符流转变为标记（Token）的过程，单个字符是程序编写时的最小字符，但标记才是编译时的最小元素。

词法分析过程由com.sun.tools.javac.parser.Scanner类来实现。

语法分析是根据标记序列构造抽象语法数的过程，抽象语法树（Abstract Syntax Tree AST）是一种用来描述程序代码语法结构的树形表示方式，抽象语法树的每一个节点都代表着程序代码中的一个语法结构（Syntax Construct），例如：包，类型、修饰符、运算符、接口、返回值甚至代码注释都可以是一种特定的语法结构。

Java语法树由com.sun.tools.javac.tree.JCTree实现。

填充符号表 - 处理过程 1.2 enterTrees

符号表（Symbal Table）是由一组符号地址和符号信息构成的数据结构。

填充符号表的过程由com.sun.tools.javac.comp.Enter类实现，该过程的产出物是一个待处理列表，其中包含了每一个编译单元的抽象语法树的顶级节点。

注解处理器 - 处理过程 2 processAnnotations

插入式注解处理器可以看成一组编译器的插件，当这些插件工作时，允许读取、修改、添加抽象语法树的任意元素。如果插件对语法树进行了修改，编译器将回到解析阶段重新处理，直到所有插入式注解处理器都执行完修改，每一次循环称为一个轮次（Round）。

案例：

  • Lombok
  • CheckStyle
  • FindBugs

语义分析与字节码生成 - 处理过程 3

标注检查 - 处理过程 3.1 attribute

标注检查内容：
  • 变量使用前是否已声明
  • 变量与赋值之间的数据类型是否能匹配
  • ………………

标注检查过程还会做一个常量折叠（Constant Folding）的优化。

int i = 1 + 2;

抽象语法树表示为 ConstantExpressionValue:3

数据及控制流分析 - 处理过程 3.2 flow

数据流分析和控制流分析是对程序的上下文进行进一步的验证：
  • 程序局部变量在使用前是否有赋值
  • 方法的每条路径是否有返回值
  • 是否所有的受查异常都被正确处理
  • ………………

解语法糖 - 处理过程 3.3 desugar

Java常见语法糖：
  • 泛型
  • 变长参数
  • 自动装箱、拆箱
  • foreach
  • ………………

Java虚拟机运行时并不支持这些语法，它们在编译阶段被还原回原始的基础语法结构。

字节码生成 - 处理过程 3.4 generate

字节码生成是Javac编译过程的最后一个阶段，在Javac源码里面由com.sun.tools.javac.jvm.Gen类来完成。

字节码生成阶段不仅仅是把前面各个步骤所生成的信息（语法树、符号表）转化为字节码指令写到磁盘中，编译器还进行了少量的代码添加和转换工作。
